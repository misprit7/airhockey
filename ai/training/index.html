Training a robotic air hockey agent in simulation using TD-MPC2 (model-based RL with learned world model + MPPI planning), then transferring to a physical cable-driven parallel robot. Supports both curriculum training (4 stages) and simple mode (--simple) for continuous training against an idle opponent.

Why TD-MPC2?

Model-based RL learns a world model (dynamics + reward predictor) and uses MPPI planning at inference. This gives better sample efficiency than model-free methods (SAC/PPO) and natural multi-step lookahead for trajectory planning.

Why Curriculum?

Air hockey is hard: sparse goals, continuous control, fast dynamics. A 4-stage curriculum progressively introduces reward complexity and opponent difficulty, ensuring the agent builds foundational skills before facing competitive play.

4-Stage Curriculum

Each stage builds on the previous, adding reward components and increasing opponent difficulty. Stages auto-advance when a reward plateau is detected. Episode lengths grow with stage complexity. Key design choices: no proximity reward (in curriculum), forward-only contact rewards (puck must move toward opponent), shaping decays to 0 in last 40%, goal penalty ramps 0→full over first 30%, contact cap 5/ep, goals +160/-20 (game) / +130/-20 (self-play), no entropy bonus in any stage. Alternatively, use --simple mode for flat training against an idle opponent.

Plateau Detection & Stage Advancement

Stages auto-advance when the PlateauDetector identifies a reward plateau. It compares recent episodes against a per-stage lookback window: S1: 5, S2: 8, S3: 10, S4: 15 episodes (longer stages get more lookback). If improvement is below 5%, the agent is ready for the next stage.

if (recent_avg - older_avg) / |older_avg| < threshold (0.05)
   AND steps >= 150,000
   AND (stage == 1 OR avg_goals >= 1.0):
    advance to next stage
    save checkpoint, reset LR, switch opponent

Minimum 150,000 steps per stage prevents premature advancement. Episode staggering: parallel envs have different episode lengths (randomized ±10%) so episodes end at different times, producing smooth TensorBoard curves instead of periodic spikes.

Per-Stage Learning Rate Schedule

Each stage starts at a stage-specific LR and decays via cosine annealing to LR/5 over ~450k estimated steps. On stage transition, the LR resets to the new stage's initial value. The shallower decay (LR/5 vs LR/10) prevents the LR from dropping too low before plateau detection triggers advancement.

Dashed vertical lines = stage transitions (LR resets). Cosine decay within each stage from initial LR to LR/5 over ~450k steps. Encoder LR is scaled by 0.3× (slower feature learning).

Linear Reward Annealing

In the last 40% of each game stage, all shaping reward weights decay linearly to 0. This forces the agent to rely on goal rewards only by the end of each stage, preventing shaping-reward dependence before advancing.

Penalty Ramp

Goal penalty linearly ramps from 0 to full over the first 30% of each game stage. This lets the agent learn to score goals first before being penalized for conceding, preventing an early avoidance phase.

Step-Based Metrics Only

All metrics are tracked per 10k steps only: goals scored/conceded, contacts, win rate, episodes completed. Per-episode TensorBoard logging has been removed to reduce noise and provide consistent monitoring independent of episode length changes across stages.

Puck-Stuck Reset

If the puck speed stays below 0.05 for 120 consecutive steps (2 seconds), the puck is reset to center with a random direction. This prevents episodes from stalling when the puck gets wedged or the agent fails to engage.

Fixed Benchmark Eval

Every 500k steps, the agent plays 5 games each against idle, goalie, and follower opponents. A composite score tracks progress consistently across stages, catching regressions that within-stage reward curves would miss.

World Model Diagnostics

Per-step prediction error of the world model is logged to TensorBoard. This reveals when the dynamics model is struggling with new opponent behaviors or unseen game states, enabling targeted debugging.

TD-MPC2 Architecture

World Model

TD-MPC2 learns a latent-space world model with 5 heads: encoder, dynamics (MLP), reward, termination, and policy. Q-values are estimated by a 5-member ensemble. The MLP dynamics model predicts the next latent state from the current state and action — simple, fast, and 14× lower prediction error than GRU (0.0009 vs 0.013).

MLP Dynamics — Simple Beats Recurrent

The dynamics model uses a standard feedforward MLP. GRU-based dynamics was tested and reverted — the MLP achieves 14× lower prediction error (0.0009 vs 0.013) while being simpler and faster. Velocities are provided directly in the observation, so temporal memory is unnecessary.

ComponentDetail

Why MLP over GRU? With explicit velocities in the 14-dim observation, the encoder already captures full state information. The GRU's hidden state added complexity (per-env tracking, MPPI branching, episode boundary resets) without improving prediction quality — it actually hurt it by 14×.

Why not Transformer? Also tested and removed. At our scale (512-dim latent, 2-dim action), MLP is faster than self-attention for horizons ≤30.

  # MLP dynamics forward pass
  z_a = torch.cat([z, action], dim=-1)   # concat latent + action
  z_next = dynamics_mlp(z_a)             # MLP → next latent
  z_next = sim_norm(z_next)              # SimNorm → next latent z'

MPPI Planning (Batched)

At inference, MPPI samples 512 action trajectories over a 5-step horizon with 6 refinement iterations, evaluates them using the world model's Q-values, and selects the best action. This runs in parallel across all 64 environments in a single batched forward pass.

Animated: gray = sampled trajectories, colored = elite trajectories, green = selected action

Prioritized Experience Replay (PER)

The replay buffer uses prioritized sampling based on TD-error magnitude. High-error transitions (surprising outcomes) are replayed more frequently, accelerating learning on difficult scenarios.

Priority exponent (α)0.6
IS exponent (β)0.4 → 1.0
β annealingLinear over training
Min priority (ε)1e-6

β annealing corrects for the bias introduced by non-uniform sampling. It starts at 0.4 (allowing biased but fast early learning) and linearly increases to 1.0 (fully unbiased) by the end of training, ensuring convergence.

Training Loop

for step in range(total_steps):
  # 1. Plan actions (batched MPPI)
  actions = agent.act(obs, t0=reset_mask)

  # 2. Step all N envs
  obs, rewards, done = batch_env.step(actions)

  # 3. Shape rewards (curriculum stage)
  shaped = reward_shaper.compute(obs, rewards)

  # 4. Store trajectories in PER buffer
  buffer.add(trajectories)

  # 5. Gradient update (1 update/step)
  batch, weights = buffer.sample(prioritized=True)
  td_errors = agent.update(batch, weights)
  buffer.update_priorities(td_errors)
  lr_scheduler.step()

  # 6. Check plateau, advance stage

Batched MPPI

Instead of planning separately for each env, MPPI trajectories for all 64 envs are evaluated in a single batched forward pass through the world model. This amortizes GPU kernel launch overhead.

torch.compile

torch.compile on the gradient update (_update) gives 1.9-3.4x speedup with a one-time ~33s warmup. Not applied to planning (graph breaks from randn/topk).

Large Batch Updates

Batch size 8192 (vs default 256) with 1 update per iteration. GPU scaling is sub-linear: a 32× batch increase yields ~15× throughput gain, since kernel overhead is amortized across more data per launch.

Why Explicit Velocities Over Frame Stacking?

Frame stacking (5 position-only frames = 32 dims) was tested but reverted in favor of explicit velocities (14 dims). Direct velocity observations proved critical for learning quality — the world model error dropped 14× with MLP dynamics on velocity observations vs GRU dynamics on stacked frames. The simpler representation also makes the observation space smaller, the encoder's job easier, and enables a stateless MLP dynamics model.

Self-Play System

Stage 4 uses an Elo-rated opponent pool to prevent cyclical forgetting and promote continual improvement. Current run (selfplay_v15): 2.15M steps, 81% win rate vs 43 opponent versions.

Opponent Pool

Up to 100 snapshots of past agent checkpoints are stored on CPU. New snapshots are added every 50k steps. Old snapshots are pruned using a mix of recency, diversity, and difficulty.

Sampling: Opponents are selected based on match quality — prefer opponents near the current agent's skill level for maximum learning signal.

Inference: Opponents use full MPPI planning, same as the training agent. This ensures realistic opponent behavior and proper competitive pressure, at the cost of higher compute per step.

Elo Rating Distribution

Each snapshot tracks win/loss statistics. Elo ratings approximate relative skill levels.

Example pool: older snapshots at lower Elo, recent ones approaching the current agent's skill.

Why Self-Play?

Scripted opponents have a skill ceiling. Once the agent beats the follower consistently, there's nothing harder to train against. Self-play creates an endlessly escalating challenge where the opponent improves alongside the agent. Dropping all auxiliary rewards in Stage 4 ensures the agent optimizes for pure game-winning ability without relying on reward-shaping crutches. Score handicaps (30% of episodes start with non-zero scores) prevent the agent from only learning to play from 0-0.

Game Recording

Periodically during training, a synchronous 30-second recording is run on the main thread. An earlier async approach using separate CPU/CUDA streams was removed in favor of this simpler synchronous method. Recordings are saved in columnar JSON and viewable in the web UI for debugging agent behavior at each stage. Puck-stuck reset: if puck speed stays below 0.05 for 120 steps (2 seconds), the puck resets to center with a random direction.

Lessons Learned

Key failures from earlier training iterations and the fixes applied.

Shaping Rewards Dominated Goals

Problem: With higher shaping weights (contact 1.0, directed 0.5, placement 0.5) and lower goal rewards (+80/-40), auxiliary rewards contributed more total reward than goals. The agent optimized for hitting and shaping the puck rather than actually scoring.

Fix: Cut all shaping weights drastically (contact 0.1, directed 0.1, defense 0.1, progress 0.1, placement 0.2). Removed proximity entirely. Added linear reward annealing — shaping decays to 0 in the last 40% of each stage. Added contact cap (max 5 per episode) and penalty ramp (0→full over first 30%). Boosted goal reward to +160/-20 so goals dominate the signal.

Agent Rushing Without Scoring

Problem: The agent advanced stages by plateauing on shaping rewards without ever learning to score goals. It could chase and hit the puck consistently but couldn't convert hits into goals.

Fix: The min goals gate (1+ goals/ep) blocks advancement until scoring ability is proven. Combined with reduced shaping weights, goal rewards now dominate the signal. The Corner stage was removed entirely — it added training time without teaching useful skills beyond what Goalie provides.

No Fixed Benchmark

Problem: Progress was only measured by within-stage reward curves, which made it hard to compare across stages or detect regressions. Reward scale changes between stages masked actual skill degradation.

Fix: Added fixed benchmark evaluation every 500k steps. The agent plays 5 games each against idle, goalie, and follower opponents with a composite score, providing a consistent skill metric across all stages.

Self-Play Fragility

Problem: The agent learned to play well from a 0-0 start but collapsed when behind. In real games, falling behind early led to increasingly poor play as the agent had no experience recovering from a deficit.

Fix: Added score handicaps in self-play: 30% of episodes start with non-zero scores (random advantage/disadvantage). This forces the agent to learn comeback strategies and to maintain composure when ahead.

GRU and Transformer Dynamics: Both Worse Than MLP

Both tested and reverted. GRU dynamics had 14× higher world model error (0.013 vs 0.0009) compared to a simple MLP. With explicit velocities in the observation, temporal memory is unnecessary — the full state is available at every timestep. Transformer-based dynamics was also benchmarked but at our scale (512-dim latent, 2-dim action), self-attention overhead far exceeds any benefit for horizons ≤30. Simple MLP wins on both accuracy and speed.
